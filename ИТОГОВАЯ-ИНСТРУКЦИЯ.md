# Итоговая инструкция - Dell R760 AI Server

## 🎯 Главное

**Проблема:** Вы находитесь НЕ в локальной сети с сервером (192.168.1.x)

**Решение:** Нужен внешний IP адрес для удаленного подключения

## 📞 Что попросить у человека на месте

### 1. Узнать внешний IP адрес

Попросите открыть на любом устройстве в той же сети с сервером:
- **https://2ip.ru**
- **https://myip.ru**

Или выполнить в терминале:
```bash
curl ifconfig.me
```

**Пример внешнего IP:** 198.51.100.1

### 2. Настроить сеть на сервере

Попросите подключить монитор к серверу и выполнить:

**Вход:**
- Username: `youruser`
- Password: `***REDACTED***`

**Команды (скопировать весь блок):**
```bash
# Проверка сети
ip addr show
ip route show
ping -c 3 192.168.1.1

# Если сеть не работает, узнайте имя интерфейса
ip link show

# Настройка IP (замените eno1 на ваш интерфейс)
sudo ip addr add YOUR_LAN_HOST/24 dev eno1
sudo ip link set eno1 up
sudo ip route add default via 192.168.1.1
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf

# Проверка
ping -c 3 192.168.1.1
ping -c 3 8.8.8.8

# Настройка SSH
sudo systemctl start ssh
sudo systemctl enable ssh
```

### 3. Установить AI модели на сервере

После настройки сети, выполните на сервере:

```bash
# Полная установка (одной командой)
curl -fsSL https://ollama.com/install.sh | sh && \
sudo mkdir -p /etc/systemd/system/ollama.service.d && \
echo '[Service]' | sudo tee /etc/systemd/system/ollama.service.d/override.conf && \
echo 'Environment="OLLAMA_HOST=0.0.0.0:11434"' | sudo tee -a /etc/systemd/system/ollama.service.d/override.conf && \
sudo systemctl daemon-reload && \
sudo systemctl restart ollama && \
sudo systemctl enable ollama && \
sudo apt install -y nvidia-driver-535 nvidia-utils-535 && \
sudo apt install -y ufw && \
sudo ufw allow 22/tcp && \
sudo ufw allow 11434/tcp && \
sudo ufw --force enable && \
echo "Установка завершена! Требуется перезагрузка." && \
echo "Выполните: sudo reboot"
```

**Перезагрузка:**
```bash
sudo reboot
```

**После перезагрузки (войдите снова):**
```bash
# Проверка GPU
nvidia-smi

# Загрузка моделей (30-60 минут!)
ollama pull qwen2.5:72b
ollama pull llama3.2-vision:90b
ollama pull deepseek-coder:33b

# Проверка
ollama list

# Тест
ollama run qwen2.5:72b "Привет! Напиши Hello World."
```

## 🔧 Проброс портов в роутере

Убедитесь что в роутере (192.168.1.1) настроены правила:

| Внешний порт | Внутренний IP | Внутренний порт | Описание |
|--------------|---------------|-----------------|----------|
| 42 | YOUR_LAN_HOST | 22 | SSH |
| 11434 | YOUR_LAN_HOST | 11434 | Ollama API |

**Как проверить:**
1. Зайдите в роутер: http://192.168.1.1
2. Найдите раздел "Port Forwarding" или "NAT"
3. Проверьте что есть правило для порта 42
4. Добавьте правило для порта 11434 (если нужен API из интернета)

## 📊 Использование моделей

### После установки вы сможете:

```bash

```

**2. Через API из Windows:**
```powershell
$body = @{
    model = "qwen2.5:72b"
    prompt = "Напиши код на Python для работы с файлами"
    stream = $false
} | ConvertTo-Json

$response = Invoke-RestMethod -Uri "http://YOUR_LAN_HOST:11434/api/generate" -Method Post -Body $body -ContentType "application/json"
Write-Host $response.response
```

**3. Через Python клиент:**
```bash
python ollama-client.py ask "qwen2.5:72b" "Привет! Объясни что такое нейросети."
```

**4. Vision модель (анализ изображений):**
```bash
python ollama-client.py vision "llama3.2-vision:90b" "Опиши изображение" photo.jpg
```
