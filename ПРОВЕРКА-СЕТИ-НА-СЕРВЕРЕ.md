## ШАГ 2: Проверка сетевых интерфейсов

```bash
# Показать все сетевые интерфейсы
ip addr show

# Или
ifconfig -a
```

**Что искать:**
- Интерфейс должен иметь IP адрес YOUR_LAN_HOST
- Статус должен быть UP
- Пример: `inet YOUR_LAN_HOST/24`

---

## ШАГ 3: Если IP адреса нет или неправильный

### Вариант А: Быстрая настройка (временная)

```bash
# Узнайте имя интерфейса (обычно eno1, eth0, enp0s1)
ip link show

# Предположим интерфейс называется eno1
# Настройте IP вручную:
sudo ip addr add YOUR_LAN_HOST/24 dev eno1
sudo ip link set eno1 up
sudo ip route add default via 192.168.1.1

# Проверка
ip addr show eno1
ping 192.168.1.1
```

### Вариант Б: Постоянная настройка через Netplan

```bash
# Узнайте имя интерфейса
ip link show

# Создайте/отредактируйте конфигурацию
sudo nano /etc/netplan/00-installer-config.yaml
```

**Содержимое файла:**
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eno1:
      addresses:
        - YOUR_LAN_HOST/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
```

**ВАЖНО:** Замените `eno1` на имя вашего интерфейса!

**Применить настройки:**
```bash
sudo netplan apply
```

**Проверка:**
```bash
ip addr show
ping 192.168.1.1
ping 8.8.8.8
```

## ШАГ 4: Проверка SSH сервиса

```bash
# Проверка статуса SSH
sudo systemctl status ssh

# Если не запущен
sudo systemctl start ssh
sudo systemctl enable ssh

# Проверка что SSH слушает порт 22
sudo netstat -tulpn | grep :22

# Или
sudo ss -tulpn | grep :22
```

---

## ШАГ 5: Проверка файрвола

```bash
# Проверка статуса UFW
sudo ufw status

# Если файрвол блокирует SSH
sudo ufw allow 22/tcp
sudo ufw reload

# Или временно отключите файрвол для теста
sudo ufw disable
```

---

## ШАГ 6: Тест подключения изнутри

```bash
# Проверка что SSH работает локально
ssh localhost

# Проверка что сервер слушает на всех интерфейсах
sudo nano /etc/ssh/sshd_config
```

Убедитесь что есть строка:
```
ListenAddress 0.0.0.0
```

Если нет, добавьте и перезапустите SSH:
```bash
sudo systemctl restart ssh
```

## Если сеть работает, сразу запустите установку

```bash
# Загрузите скрипт
wget https://raw.githubusercontent.com/ollama/ollama/main/scripts/install.sh -O /tmp/install-ollama.sh

# Или создайте файл вручную
nano ~/quick-setup.sh
```

Вставьте в файл:
```bash
#!/bin/bash
set -e

echo "=== Quick AI Setup ==="

# Update
sudo apt update

# NVIDIA drivers
sudo apt install -y nvidia-driver-535 nvidia-utils-535

# Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER

# Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Configure Ollama
sudo mkdir -p /etc/systemd/system/ollama.service.d
echo '[Service]' | sudo tee /etc/systemd/system/ollama.service.d/override.conf
echo 'Environment="OLLAMA_HOST=0.0.0.0:11434"' | sudo tee -a /etc/systemd/system/ollama.service.d/override.conf
sudo systemctl daemon-reload
sudo systemctl restart ollama
sudo systemctl enable ollama

# Firewall
sudo apt install -y ufw
sudo ufw allow 22/tcp
sudo ufw allow 11434/tcp
sudo ufw --force enable

echo "=== Setup complete! Reboot required ==="
echo "Run: sudo reboot"
```

Запустите:
```bash
chmod +x ~/quick-setup.sh
./quick-setup.sh
```

После перезагрузки:
```bash
sudo reboot
```

Затем загрузите модели:
```bash
ollama pull qwen2.5:72b
ollama pull llama3.2-vision:90b
```
