# Решение проблемы подключения

## Ситуация

❌ Не работает подключение к:
- `YOUR_LAN_HOST` (сервер)
- `192.168.0.120` (iDRAC)

**Причина:** Вы находитесь НЕ в локальной сети с сервером.

---

## Варианты решения

### Вариант 1: Использовать внешний IP адрес (через интернет)

Вы уже настроили проброс порта 42 → YOUR_LAN_HOST:22 в роутере.

**Нужно узнать внешний IP адрес роутера:**

#### Способ А: Попросите человека у сервера узнать внешний IP

Пусть откроют на любом устройстве в той же сети:
- https://myip.ru
- https://2ip.ru
- https://whatismyip.com

Или выполнят команду:
```bash
curl ifconfig.me
curl icanhazip.com
```

#### Способ Б: Проверьте настройки роутера

Если у вас есть доступ к роутеру (192.168.1.1) через VPN или другой способ:
1. Зайдите в роутер
2. Найдите раздел "WAN" или "Интернет"
3. Посмотрите внешний IP адрес

---

### Вариант 2: VPN к локальной сети

Настройте VPN на роутере для доступа к локальной сети 192.168.1.x

**Популярные варианты:**
- WireGuard (современный, быстрый)
- OpenVPN (универсальный)
- L2TP/IPSec (встроен в Windows)

После настройки VPN вы сможете подключаться к YOUR_LAN_HOST напрямую.

---

### Вариант 3: TeamViewer / AnyDesk на компьютер в той же сети

Если есть компьютер в локальной сети с сервером:
1. Установите TeamViewer или AnyDesk
2. Подключитесь к этому компьютеру удаленно
3. С него подключайтесь к серверу по SSH

## 📋 Инструкция для человека у роутера

Попросите выполнить:

1. **Узнать внешний IP:**
   - Открыть https://2ip.ru
   - Записать IP адрес (например: 85.192.45.123)
   - Прислать вам этот IP

2. **Проверить проброс портов в роутере:**
   - Зайти в роутер: http://192.168.1.1
   - Найти раздел "Port Forwarding" или "NAT"
   - Убедиться что есть правило: порт 42 → YOUR_LAN_HOST:22
   - Добавить правило: порт 11434 → YOUR_LAN_HOST:11434

3. **Проверить что сервер в сети:**
   - Открыть командную строку
   - Выполнить: `ping YOUR_LAN_HOST`
   - Должен отвечать

4. **Подключиться к серверу локально:**
   ```bash
   ssh user@YOUR_LAN_HOST
   ```
   Пароль: `REDACTED`

5. **Выполнить команды из файла `commands-for-server.txt`**

## 🚀 После получения внешнего IP

Предположим внешний IP: **85.192.45.123**

### Подключение с вашего компьютера:


### Установка AI моделей:

После успешного подключения выполните на сервере:

```bash
# Быстрая установка (одной командой)
curl -fsSL https://ollama.com/install.sh | sh && \
sudo mkdir -p /etc/systemd/system/ollama.service.d && \
echo '[Service]' | sudo tee /etc/systemd/system/ollama.service.d/override.conf && \
echo 'Environment="OLLAMA_HOST=0.0.0.0:11434"' | sudo tee -a /etc/systemd/system/ollama.service.d/override.conf && \
sudo systemctl daemon-reload && \
sudo systemctl restart ollama && \
sudo systemctl enable ollama && \
sudo apt install -y nvidia-driver-535 && \
sudo apt install -y ufw && \
sudo ufw allow 22/tcp && \
sudo ufw allow 11434/tcp && \
sudo ufw --force enable
```

Затем перезагрузка:
```bash
sudo reboot
```

После перезагрузки подключитесь снова и загрузите модели:
```bash
ssh -p 42 user@85.192.45.123

# Загрузка моделей (30-60 минут)

ollama pull llama3.2-vision:90b
ollama pull deepseek-coder:33b

# Проверка
```

## 💡 Важно понимать

**Локальная сеть (192.168.x.x)** - доступна только внутри здания где стоит сервер.

**Внешний IP** - доступен из любой точки мира через интернет.

Для удаленного доступа нужен либо:
- Внешний IP + проброс портов (уже настроен порт 42)
- VPN к локальной сети
- Удаленный рабочий стол (TeamViewer) на компьютер в той сети
