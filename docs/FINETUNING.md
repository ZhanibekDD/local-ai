# Первый цикл SFT: модель, датасеты, объём

Рекомендуемая база для **первого** цикла дообучения: **`Qwen/Qwen2.5-14B-Instruct`**.

## Почему эта модель

- Это **instruction**-модель из линейки **Qwen2.5** (лучше instruction following, structured outputs/JSON, заявлена поддержка многих языков, включая русский).
- **14B** — разумный старт для **A40 48 GB**: можно быстро прогнать несколько итераций без «вечных» экспериментов.
- **Не начинать с 72B**: имеет смысл, когда датасет уже чистый, пайплайн обучения проверен, eval показывает, что дообучение реально помогает. Сразу 72B — дорого и нервно, не «умно и практично». У Qwen2.5 есть размеры от 0.5B до 72B; **14B** — нормальный стартовый полигон.

## Датасеты с Hugging Face

Разумный набор: **2 публичных + 1 свой**.

### 1) `HuggingFaceH4/no_robots`

- Человеческие пары, не «чистая GPT-синтетика».
- Порядка **10 000** instruction–response примеров, ориентир на **SFT**.
- **Зачем:** лучше следование инструкциям, меньше лишней болтовни, стабильнее формат.

### 2) `HuggingFaceTB/OpenHermes-2.5-H4`

- Удобная H4-версия OpenHermes с полем `messages` и split **`train_sft`** (в карточке — порядка **~951k** строк).
- **Зачем:** большой instruction/chat-корпус.
- **Важно:** не тащить **весь** датасет в первый прогон — он крупный; в описании исходной линейки фигурирует порядок **~1M+** строк синтетической компиляции. На первом круге — **только сэмпл** (см. ниже).

### 3) Собственный датасет (главный слой)

JSONL из **реальных** задач проекта, например:

- русский чат;
- «коротко и по делу»;
- строгий JSON;
- OCR text → JSON;
- классификация, support, бизнес-сценарии.

Публичные датасеты — разминка; **свой** слой даёт рост именно под ваш продукт.

## Рекомендуемый состав первого SFT

| Пакет | Источник | Объём на первый цикл |
|--------|-----------|----------------------|
| Instruction quality | `no_robots` | **весь** (~10k) |
| General instruction/chat | OpenHermes-2.5-H4 (`train_sft`) | **сэмпл 20k–60k**, не весь миллион |
| Доменный | свой JSONL | **2k–10k** хороших примеров (даже **500–2000** уже полезны, если качество высокое) |

**Итог первого круга:** ориентир **~40k–45k** примеров (например ~10k + ~30k сэмпл + ~2k–5k своих) — разумный масштаб для проверки пайплайна.

## Чего не делать

- Не заливать «любой русский датасет» без критериев качества и задачи.
- Не брать **весь** OpenHermes (~1M+ строк) в первый прогон.
- Не смешивать код, OCR, vision и общий чат в один неструктурированный «суп».
- Не учить на **сырых** Telegram-логах без очистки и разметки.

## Что учим на первом этапе

Только **SFT** (supervised fine-tuning). **DPO** — позже, когда базовый SFT и eval уже стабильны.

Инструменты (ориентиры из экосистемы Qwen и HF):

- **LLaMA-Factory** — в документации Qwen часто указывают SFT через него, в т.ч. **LoRA / Q-LoRA / DoRA**.
- **TRL `SFTTrainer`** — поддержка conversational-датасетов и chat templates для instruct-моделей.

## Файлы и скрипты в этом репозитории

| Что | Где |
|-----|-----|
| Формат и примеры JSONL | [data/sft/README.md](../data/sft/README.md), `data/sft/custom.example.jsonl` |
| Валидация строк | `python training/validate_sft_jsonl.py data/sft/custom.example.jsonl` |
| Слияние нескольких JSONL + shuffle | `python training/merge_jsonl.py a.jsonl b.jsonl -o data/sft/build/merged.jsonl --seed 42` |
| Сэмпл с Hugging Face | `pip install -e ".[train]"` затем `python training/sample_hf_dataset.py ...` (см. хелп скрипта) |

Примеры выгрузки:

```bash
pip install -e ".[train]"

python training/sample_hf_dataset.py --dataset HuggingFaceH4/no_robots --split train \
  -o data/sft/build/no_robots.jsonl

python training/sample_hf_dataset.py --dataset HuggingFaceTB/OpenHermes-2.5-H4 --split train_sft \
  --max-samples 30000 --seed 42 -o data/sft/build/openhermes_30k.jsonl
```

Каталог `data/sft/build/` в `.gitignore` — туда складывайте сгенерированные большие файлы. Дальше: LLaMA-Factory или TRL на машине с GPU, указав путь к объединённому JSONL и базовую модель `Qwen/Qwen2.5-14B-Instruct`.
