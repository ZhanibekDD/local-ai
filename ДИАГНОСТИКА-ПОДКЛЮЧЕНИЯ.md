# Диагностика подключения к серверу Dell R760

## Проблема
Сервер не отвечает на SSH подключения с IP YOUR_LAN_HOST

---

## Возможные причины

1. **Сервер выключен или не загружен**
2. **Неправильный IP адрес**
3. **SSH сервис не запущен на сервере**
4. **Файрвол блокирует подключения**
5. **Сервер в другой подсети**

---

## Шаги диагностики

### 1. Проверка физического подключения

- Убедитесь что сервер включен (индикаторы питания горят)
- Проверьте что сетевой кабель подключен
- Проверьте индикаторы сетевых портов (должны мигать)

### 2. Проверка через iDRAC9

Ваш сервер имеет iDRAC9 Enterprise - это удаленное управление.

**Подключение к iDRAC:**
1. Откройте браузер
2. Перейдите на один из адресов:
   - `https://YOUR_LAN_HOST` (если iDRAC на том же IP)
   - `https://192.168.0.120` (стандартный IP iDRAC)
   - Проверьте наклейку на сервере с IP адресом iDRAC

**Через iDRAC можно:**
- Проверить статус сервера (включен/выключен)
- Посмотреть консоль сервера
- Узнать текущий IP адрес
- Включить/выключить сервер удаленно
- Открыть виртуальную консоль (Virtual Console)

**Стандартные учетные данные iDRAC:**
- Username: `root`
- Password: `<BMC password per vendor>` (или указан на наклейке сервера)

### 3. Проверка сети из Windows

Откройте PowerShell и выполните:

```powershell
# Проверка ARP таблицы (найти MAC адрес сервера)
arp -a

# Сканирование сети (найти все устройства)
# Установите nmap если нужно
nmap -sn 192.168.1.0/24

# Или используйте встроенную команду
1..254 | ForEach-Object {
    $ip = "192.168.1.$_"
    if (Test-Connection -ComputerName $ip -Count 1 -Quiet) {
        Write-Host "Найден: $ip"
    }
}
```

### 4. Прямое подключение к серверу

Если есть физический доступ к серверу:

1. Подключите монитор и клавиатуру к серверу
2. Войдите в систему (youruser / ***REDACTED***)
3. Выполните команды:

```bash
# Проверка сетевых интерфейсов
ip addr show

# Проверка SSH сервиса
sudo systemctl status ssh

# Если SSH не запущен
sudo systemctl start ssh
sudo systemctl enable ssh

# Проверка файрвола
sudo ufw status

# Если файрвол блокирует SSH
sudo ufw allow 22/tcp
sudo ufw reload

# Проверка слушающих портов
sudo netstat -tulpn | grep :22
```

### 5. Альтернативные способы подключения

#### Вариант A: Через проброшенный порт 42
```bash

```

#### Вариант Б: Через iDRAC Virtual Console

2. Откройте Virtual Console
3. Работайте с сервером как с физической консолью

#### Вариант В: Через последовательный порт (Serial)

## Настройка SSH на сервере (если нужно)

Если SSH не настроен, выполните на сервере:

```bash
# Установка OpenSSH Server
sudo apt update
sudo apt install -y openssh-server

# Запуск и автозапуск
sudo systemctl start ssh
sudo systemctl enable ssh

# Проверка статуса
sudo systemctl status ssh

# Настройка для приема подключений
sudo vim /etc/ssh/sshd_config
```

Убедитесь что в `/etc/ssh/sshd_config`:
```
Port 22
ListenAddress 0.0.0.0
PermitRootLogin yes
PasswordAuthentication yes
```

Перезапуск SSH:
```bash
sudo systemctl restart ssh
```

## Проверка сетевой конфигурации

На сервере выполните:

```bash
# Текущий IP адрес
ip addr show

# Таблица маршрутизации
ip route show

# DNS
cat /etc/resolv.conf

# Проверка шлюза
ping -c 3 192.168.1.1

# Проверка интернета
ping -c 3 8.8.8.8
```

### Настройка статического IP (если нужно)

Если IP адрес изменился, настройте статический:

```bash
# Найдите имя интерфейса
ip link show

# Предположим интерфейс называется eno1
sudo vim /etc/netplan/00-installer-config.yaml
```

Содержимое:
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eno1:
      addresses:
        - YOUR_LAN_HOST/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
```

Применение:
```bash
sudo netplan apply
```
