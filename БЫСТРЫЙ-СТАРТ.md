## Быстрая установка (копируйте блоками)

### 1. Обновление и базовые инструменты
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential git wget curl vim htop tmux net-tools python3-pip
```

### 2. NVIDIA драйверы
```bash
sudo apt install -y nvidia-driver-535 nvidia-utils-535
sudo reboot
```

**После перезагрузки подключитесь снова и проверьте:**
```bash
nvidia-smi
```

### 3. Docker
```bash
curl -fsSL https://get.docker.com -o /tmp/get-docker.sh
sudo sh /tmp/get-docker.sh
sudo usermod -aG docker $USER
```

### 4. NVIDIA Container Toolkit
```bash
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt update
sudo apt install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

**Проверка:**
```bash
docker run --rm --gpus all nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi
```

### 5. Ollama
```bash
curl -fsSL https://ollama.com/install.sh | sh

sudo mkdir -p /etc/systemd/system/ollama.service.d
sudo tee /etc/systemd/system/ollama.service.d/override.conf > /dev/null <<EOF
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment="OLLAMA_ORIGINS=*"
EOF

sudo systemctl daemon-reload
sudo systemctl restart ollama
sudo systemctl enable ollama
systemctl status ollama
```

### 6. Файрвол
```bash
sudo apt install -y ufw
sudo ufw allow 22/tcp
sudo ufw allow 11434/tcp
sudo ufw --force enable
sudo ufw status
```

### 7. Загрузка моделей (20-60 минут)
```bash
# Текстовая модель (лучшая для русского)
ollama pull qwen2.5:72b

# Vision модель
ollama pull llama3.2-vision:90b

# Модель для кода
ollama pull deepseek-coder:33b

# Проверка
ollama list
```

## Использование моделей

### Интерактивный режим:
```bash
# Текстовая модель
ollama run qwen2.5:72b
>>> Привет! Напиши функцию сортировки на Python.
>>> /bye  # Выход

# Vision модель
ollama run llama3.2-vision:90b
>>> Опиши изображение /path/to/image.jpg
```

### API (из терминала сервера):
```bash
curl http://localhost:11434/api/generate -d '{
  "model": "qwen2.5:72b",
  "prompt": "Привет! Как дела?",
  "stream": false
}'
```

### API (из Windows PowerShell):
```powershell
$body = @{
    model = "qwen2.5:72b"
    prompt = "Привет! Напиши код на Python."
    stream = $false
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://YOUR_LAN_HOST:11434/api/generate" -Method Post -Body $body -ContentType "application/json"
```

### Python клиент:
```python
import requests

response = requests.post('http://YOUR_LAN_HOST:11434/api/generate', json={
    'model': 'qwen2.5:72b',
    'prompt': 'Напиши Hello World на Python',
    'stream': False
})

print(response.json()['response'])
```

## Если что-то пошло не так

### Сервер не отвечает:
1. Проверьте что сервер включен
2. Подключите монитор и клавиатуру
3. Используйте iDRAC9 для удаленного управления

### GPU не работает:
```bash
sudo apt purge -y nvidia-*
sudo apt autoremove -y
sudo apt install -y nvidia-driver-535
sudo reboot
```

### Ollama не запускается:
```bash
sudo journalctl -u ollama -n 50
sudo systemctl restart ollama
```

### Нехватка памяти для модели:
```bash
# Используйте квантизованные версии
ollama pull qwen2.5:72b-q4_0
ollama pull llama3.1:70b-q4_K_M
```

## Альтернатива: Web интерфейс

Для удобного веб-интерфейса:

```bash
mkdir -p ~/ai-stack
cd ~/ai-stack

cat > docker-compose.yml <<'EOF'
version: '3.8'

services:
  ollama:
    image: ollama/ollama:latest
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    restart: unless-stopped

  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    ports:
      - "3000:8080"
    environment:
      - OLLAMA_BASE_URL=http://ollama:11434
    volumes:
      - open_webui_data:/app/backend/data
    depends_on:
      - ollama
    restart: unless-stopped

volumes:
  ollama_data:
  open_webui_data:
EOF

docker compose up -d
```

Откройте в браузере: `http://YOUR_LAN_HOST:3000`
