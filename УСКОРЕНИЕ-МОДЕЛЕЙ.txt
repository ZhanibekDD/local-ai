НАСТРОЙКА ПРОИЗВОДИТЕЛЬНОСТИ МОДЕЛЕЙ

=================================================================
  ПРОБЛЕМА: Модели работают медленно
=================================================================

Причины:
1. Модели очень большие (72B, 90B параметров)
2. Нужна оптимизация параметров
3. Возможно не используется GPU полностью


=================================================================
  РЕШЕНИЕ 1: Создать оптимизированные версии моделей
=================================================================

Скопируйте команды из файла: ОПТИМИЗАЦИЯ-МОДЕЛЕЙ.txt

Это создаст 3 новые модели:
  • qwen-ru - Qwen2.5 с русским языком и оптимизацией
  • coder-ru - DeepSeek Coder с русскими комментариями
  • vision-ru - Llama Vision с русскими описаниями

Преимущества:
  ✓ Настроенная температура для лучших ответов
  ✓ Оптимизированный контекст
  ✓ Системные промпты на русском
  ✓ Использование GPU


=================================================================
  РЕШЕНИЕ 2: Настройка Ollama для производительности
=================================================================

На сервере выполните:

# Остановка Ollama
sudo systemctl stop ollama

# Редактирование конфигурации
sudo mkdir -p /etc/systemd/system/ollama.service.d

sudo tee /etc/systemd/system/ollama.service.d/override.conf > /dev/null << 'EOF'
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment="OLLAMA_NUM_PARALLEL=2"
Environment="OLLAMA_MAX_LOADED_MODELS=1"
Environment="OLLAMA_FLASH_ATTENTION=1"
Environment="OLLAMA_KV_CACHE_TYPE=q8_0"
EOF

# Перезагрузка и запуск
sudo systemctl daemon-reload
sudo systemctl start ollama
sudo systemctl status ollama


=================================================================
  РЕШЕНИЕ 3: Использование квантизированных моделей
=================================================================

Квантизированные модели работают быстрее:

# Загрузка Q4 версий (быстрее, меньше памяти)
ollama pull qwen2.5:72b-instruct-q4_K_M
ollama pull deepseek-coder:33b-instruct-q4_K_M

# Или Q5 (баланс скорости и качества)
ollama pull qwen2.5:72b-instruct-q5_K_M


=================================================================
  РЕШЕНИЕ 4: Проверка использования GPU
=================================================================

# Мониторинг GPU в реальном времени
watch -n 1 nvidia-smi

# Проверка что модель использует GPU
ollama run qwen2.5:72b "test" &
nvidia-smi

Должно показывать:
  • GPU Utilization: 90-100%
  • Memory Used: ~45GB для Qwen 72B

=================================================================
  РЕКОМЕНДАЦИИ ПО СКОРОСТИ
=================================================================

Быстрые модели (для тестирования):
  ollama pull qwen2.5:32b        # Быстрее, 32B параметров
  ollama pull llama3.2:11b       # Очень быстрая
  ollama pull deepseek-coder:6.7b # Быстрая для кода

Для production (текущие модели):
  • Используйте оптимизированные версии (qwen-ru, coder-ru)
  • Настройте OLLAMA_MAX_LOADED_MODELS=1
  • Используйте OLLAMA_FLASH_ATTENTION=1


=================================================================
  ОЖИДАЕМАЯ СКОРОСТЬ
=================================================================

На NVIDIA A40 48GB:

Qwen2.5 72B:
  • Первый токен: 2-5 секунд
  • Генерация: 15-25 токенов/сек
  • Полный ответ (200 слов): 10-20 секунд

DeepSeek Coder 33B:
  • Первый токен: 1-3 секунды
  • Генерация: 25-40 токенов/сек

Llama Vision 90B:

  • Анализ: 15-30 секунд
  • Полное описание: 20-40 секунд
