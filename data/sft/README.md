# Данные для SFT

## Формат строки (JSONL)

Одна строка = один JSON-объект. Поддерживаются варианты:

1. **Чат (предпочтительно для Qwen / TRL / LLaMA-Factory)**  
   Поле `messages`: список `{"role": "user"|"assistant"|"system", "content": "..."}`.  
   Опционально корневое поле `system` — его можно смержить в system-сообщение в тренере.

2. **Упрощённый шаблон**  
   См. `template.jsonl` — минимальный пример.

Файл **`custom.example.jsonl`** — эталон доменных сценариев (русский чат, JSON, OCR→JSON, классификация, support). Копируйте в свой файл (например `custom.jsonl`) и наполняйте; большие сырые выгрузки лучше класть в `build/` (см. ниже).

## Каталоги

| Путь | Назначение |
|------|------------|
| `custom.example.jsonl` | Пример разметки, коммитится в git |
| `build/` | Сгенерированные и объединённые JSONL (в `.gitignore`) — не коммитить |

Скрипты: каталог **`training/`** в корне репозитория.
