# Что делать сейчас - Сервер не отвечает

## Результат проверки

❌ **Сервер YOUR_LAN_HOST не отвечает:**
- Ping: не отвечает
- SSH порт 22: закрыт
- Ollama порт 11434: закрыт

---

## Возможные причины

1. **Сервер выключен** - самая вероятная причина
2. **Сервер не загружен** - идет загрузка ОС
3. **Неправильный IP адрес** - IP мог измениться
4. **Сетевой кабель не подключен**
5. **Сервер в другой подсети**

---

## Решение: 3 варианта

### Вариант 1: Использовать iDRAC9 (РЕКОМЕНДУЕТСЯ)

Ваш сервер имеет **iDRAC9 Enterprise** - это удаленное управление сервером.

**Шаги:**

1. Откройте браузер
2. Попробуйте один из адресов:
   - `https://YOUR_LAN_HOST` (если iDRAC на том же IP)
   - `https://192.168.0.120` (стандартный IP iDRAC)
   - Проверьте наклейку на задней панели сервера

3. Войдите в iDRAC:
   - **Username:** `root`
   - **Password:** `calvin` (или указан на наклейке)

4. В iDRAC вы сможете:
   - ✅ Проверить включен ли сервер
   - ✅ Включить/выключить сервер удаленно
   - ✅ Открыть виртуальную консоль (Virtual Console)
   - ✅ Узнать текущий IP адрес сервера
   - ✅ Посмотреть логи загрузки

5. Через Virtual Console:
   - Войдите в систему (dnepr / REDACTED)
   - Выполните команды из инструкции

---

### Вариант 2: Физический доступ к серверу

Если есть доступ к серверу:

1. **Проверьте индикаторы:**
   - Горят ли индикаторы питания?
   - Мигают ли индикаторы сетевых портов?

2. **Подключите монитор и клавиатуру:**
   - Войдите в систему: `dnepr` / `REDACTED`
   
3. **Проверьте сеть:**
   ```bash
   ip addr show
   ip route show
   ping 192.168.1.1
   ```

4. **Проверьте SSH:**
   ```bash
   sudo systemctl status ssh
   sudo systemctl start ssh
   ```

5. **Настройте статический IP (если нужно):**
   ```bash
   sudo nano /etc/netplan/00-installer-config.yaml
   ```
   
   Содержимое:
   ```yaml
   network:
     version: 2
     renderer: networkd
     ethernets:
       eno1:  # Замените на ваш интерфейс
         addresses:
           - YOUR_LAN_HOST/24
         routes:
           - to: default
             via: 192.168.1.1
         nameservers:
           addresses:
             - 8.8.8.8
             - 8.8.4.4
   ```
   
   Применить:
   ```bash
   sudo netplan apply
   ```

### Вариант 3: Сканирование сети

Найдите сервер в сети:

**PowerShell (Windows):**
```powershell
# Быстрое сканирование
1..254 | ForEach-Object {
    $ip = "192.168.1.$_"
    if (Test-Connection -ComputerName $ip -Count 1 -Quiet -TimeoutSeconds 1) {
        Write-Host "Found: $ip"
    }
}
```

**Или используйте Advanced IP Scanner:**
1. Скачайте: https://www.advanced-ip-scanner.com/
2. Сканируйте диапазон: 192.168.1.1 - 192.168.1.254
3. Найдите Dell сервер по MAC адресу или имени
